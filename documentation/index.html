AWS Cloud & Architectures Big Data

Description Textuelle de l'Architecture

L'architecture Big Data Serverless mise en place est conçue pour l'ingestion, le traitement et la persistance de flux massifs de données provenant de capteurs IoT en temps réel. Le point d'entrée unique est configuré via une distribution globale Amazon CloudFront couplée à une Amazon API Gateway, garantissant une ingestion sécurisée par HTTPS, une faible latence et une haute disponibilité.

Les payloads JSON entrants sont immédiatement interceptés par une unité de calcul AWS Lambda (exécutée sous Python 3.11). Cette fonction effectue une validation de schéma stricte pour rejeter les messages corrompus, calcule la température moyenne du lot et comptabilise le nombre d'anomalies détectées.

Une fois validées, les données suivent un double chemin de persistance. D'une part, les agrégats analytiques calculés sont insérés dans une base de données NoSQL Amazon DynamoDB configurée en mode de capacité à la demande pour un accès ultra-rapide. D'autre part, l'intégralité du flux brut d'origine est archivée de manière structurée et partitionnée au sein de la zone brute d'un Data Lake hébergé sur Amazon S3.
